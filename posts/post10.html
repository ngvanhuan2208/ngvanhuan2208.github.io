Advanced JavaScript – Giải mã Closures và Prototypes

JavaScript dễ học nhưng rất khó tinh thông. Closures và Prototypes chính là chìa khóa giúp lập trình viên vượt qua ranh giới Junior – Senior.

1. Closures – “Trí nhớ” kỳ diệu của hàm

Closure cho phép một hàm ghi nhớ phạm vi (scope) nơi nó được tạo ra, ngay cả khi hàm cha đã kết thúc. Đây là cơ chế giúp JavaScript tạo ra các biến riêng tư mà không cần đến class. 


function taoBoDem() {
    let count = 0;
    return function() {
        count++;
        return count;
    };
}

const counter = taoBoDem();
console.log(counter()); // 1
console.log(counter()); // 2

2. Prototypes – Cơ chế kế thừa ngầm

Phía sau cú pháp class quen thuộc, JavaScript sử dụng Prototypal Inheritance. Các phương thức được chia sẻ qua Prototype, giúp tiết kiệm bộ nhớ và tăng hiệu năng.

3. Từ khóa this – Nguồn gốc của sự nhầm lẫn

Trong JavaScript, this phụ thuộc vào cách hàm được gọi, không phải nơi nó được định nghĩa. Arrow Function không có this riêng, giúp tránh lỗi mất ngữ cảnh trong lập trình hiện đại.

4. Memory Management và Garbage Collection

JavaScript sử dụng cơ chế Mark-and-Sweep để thu hồi bộ nhớ. Các biến global hoặc event listener không được gỡ bỏ là nguyên nhân phổ biến gây rò rỉ bộ nhớ trong ứng dụng lớn.

Kết luận

Closures và Prototypes không làm JavaScript phức tạp hơn, chúng giúp ngôn ngữ này trở nên mạnh mẽ và linh hoạt. Khi làm chủ được hai khái niệm này, JavaScript sẽ hiện ra như một hệ thống logic đầy nghệ thuật.
